Create a new ADR titled: $ARGUMENTS

1. Read [templates/adr-template.md](../templates/adr-template.md).
2. Determine the next ADR number by scanning existing ADRs in the project (look in `context/adrs/`, `docs/adrs/`, or similar conventional locations).
3. Fill the template with:
   - Title from the argument above
   - Status: `Proposed`
   - Context, Decision, Consequences sections populated from the current conversation
4. Confirm the file path and content with the user before writing.

The ADR is only `Accepted` after explicit human approval.
