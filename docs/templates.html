Templates

Sixteen ready-to-use templates — the canonical shape of every artifact the framework produces.

Use them directly or via skills like create-adr, create-ctx-plan, create-exp-plan.

Plans & Specs

Context-Plan (CTX)

Context

What to read/verify, open questions, draft scope boundaries.

Feature Plan (EXP)

Exploration

Steps, affected files, risks, alternatives, acceptance criteria, Kill Criteria, Challenger.

Feature Spec

Exploration

Detailed feature specification — what the feature does, who uses it, how it integrates.

Production-Plan (PRD)

Production

Implementation plan referencing the approved EXP. Approved before any code is written.

Plan Proposal (PROP-NNN)

Any gate

Out-of-scope items captured at gates. Triaged via /promote, /defer, /reject.

Plan Registry

Index

Index of all plans in the project — status, dependencies, current step.

The Uncertainty Triplet

Mandatory before G1

Problem Statement

Who has the problem? What fails? Why now?

Hypotheses

≥2 testable hypotheses + cheapest test + kill criterion.

Risks

≥3 risks rated, top-3 expensive failure modes.

Architecture & Decisions

ADR — Architecture Decision Record

Title, Status, Context, Decision, Alternatives, Consequences. Created via /new-adr.

Architecture Invariants

Layer rules, forbidden patterns, dependency direction. Project-level extension.

Domain Patterns

Domain-specific patterns and constraints — for game, UI, data domains.

Chapter (§-numbering)

Top-level thematic area for plans — goal, scope, guiding questions. See paragraph-numbering.

Learning & Prototyping

insights.md

Required at end of Exploration. Records confirmed/killed hypotheses + production surprises. Pointers, not free-form prose.

UX Prototype

Visual reference file. Single-screen demo, sketch, or HTML mock. Required for visual/physical features.

CLAUDE.md

Project-level briefing for every AI agent — what the project is, file system overview, decisions made.
