## Long Version

AI writes code faster than ever. But speed without structure is just chaos with a compiler.

I built the chevp-ai-framework to solve a problem I kept seeing: developers letting AI generate production code without context, without a plan, and without validation. The results? Hallucinated APIs, inconsistent architectures, and code nobody truly understands.

The framework introduces a 3-step lifecycle for human-AI collaboration:

1. Context — Understand the problem before proposing solutions. Read the codebase. Gather constraints. Confirm scope with the human.
2. Exploration — Plan the solution, create specs, prototype visual work. No code until the approach is approved.
3. Production — Implement step by step, validate against the spec, deliver cleanly.

Six cross-cutting roles operate within each step: SDLC, AI-Plans, UX-Tooling, DevOps, Software-Architecture, and Context-Engineering. Quality gates between steps ensure the human approves every transition.

The core principle: AI is a tool, not an autonomous developer. It analyzes, suggests, and implements — but it does not decide. Ownership stays with the human.

The framework is open source and designed to be integrated into any project via a single CLAUDE.md file.

GitHub: https://github.com/chevp/chevp-ai-framework
