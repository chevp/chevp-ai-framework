# Prompt: AI-Framework 2-Layer Architecture Diagram

## Description

Create a technical architecture graphic that shows the 2-layer model of the AI framework.

## Prompt (for image generation)

```
Create a clean, modern technical architecture diagram on a dark background (#1a1a2e) with the following structure:

TITLE at top: "AI-Framework Architecture" in white, modern sans-serif font.

TWO MAIN LAYERS stacked vertically, each as a rounded rectangle:

BOTTOM LAYER (larger, foundational):
- Label: "chevp-ai-framework" in bold
- Subtitle: "Core — Guardrails for AI-assisted development"
- Color: deep blue gradient (#2d3a8c to #1e2761)
- Contains THREE COLUMNS inside, each as a smaller card:
  1. "Context" — icon: book/eye — "What AI must read"
  2. "Exploration" — icon: compass/iterate — "Iterative prototypes"
  3. "Production" — icon: factory/check — "Mandatory structure & validation"
- A small label at bottom-right: "generic • reusable • domain-agnostic"

TOP LAYER (sitting on top of the bottom layer):
- Label: "domain-ai-framework"
- Subtitle: "Domain-specific (e.g. Game, UI, Data Pipelines)"
- Color: teal/green gradient (#0d7377 to #14a3a8)
- Contains FOUR CARDS showing domain examples:
  1. "Game" — icons: gamepad, dice — "Scenes, NPCs, Quests, Items"
  2. "UI" — icons: layout, components — "Components, Themes, Flows"
  3. "Data" — icons: database, pipeline — "Pipelines, Transforms, Schemas"
  4. "..." — "Any domain"
- The "Game" card should be slightly highlighted/emphasized (active)

BETWEEN THE LAYERS:
- A connection arrow or bridge labeled "Submodule" showing that domain-ai-framework contains chevp-ai-framework

ABOVE THE TOP LAYER:
- A row of PROJECT BOXES showing concrete instances:
  - "nuna-ai-framework" (highlighted, connected to "Game" domain)
  - "other-ai-framework" (dimmed, connected to other domains)
- Each project box connects down to the domain layer with a thin line

AT THE VERY TOP:
- A human icon + AI icon (Claude logo style) with arrow showing:
  "Developer → AI → Framework → Artifacts"

STYLE:
- Dark theme, minimal, professional
- Rounded corners on all boxes
- Subtle shadows/glow on active elements
- Monospace font for technical labels
- Sans-serif for descriptions
- No excessive decoration
- Similar aesthetic to Vercel/Linear design systems
- Aspect ratio: 16:9 landscape
- Resolution suitable for README/documentation
```

## Usage

- README.md in project-specific frameworks
- CLAUDE.md in project monorepos
- Architecture documentation
